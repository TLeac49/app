// 01 — Fonctionnalités

Pourquoi GitHub Pages ?

01
⚡

Hébergement gratuit

GitHub Pages héberge votre site statique gratuitement, avec un domaine username.github.io.

02
🔒

HTTPS automatique

Certificat SSL/TLS généré automatiquement. Votre site est sécurisé dès le premier déploiement.

03
🔄

Chaque git push sur main déclenche un déploiement automatique via GitHub Actions.

🌐

Branchez votre propre domaine (ex: monsite.fr) en quelques minutes depuis les Settings.

// 02 — Déploiement

Mettre en ligne
en 5 étapes.

01

Créer un dépôt GitHub

Créez un nouveau repo public sur github.com. Nommez-le username.github.io pour un site principal, ou n'importe quel nom pour un sous-répertoire.

02

Initialiser et pousser votre code

Initialisez Git dans votre dossier local, ajoutez le remote et poussez votre branche main.

git init
git add .
git commit -m "initial commit"
git remote add origin https://github.com/user/repo.git
git push -u origin main
03

Activer GitHub Pages

Dans les Settings de votre repo → section Pages → Source : sélectionnez la branche main et le dossier / (root) ou /docs.

04

(Optionnel) GitHub Actions pour React/Vue

Si vous utilisez un framework avec un build step, créez un fichier workflow pour automatiser le build et le déploiement.

# .github/workflows/deploy.yml
on: push
branches: [main]
jobs:
deploy:
runs-on: ubuntu-latest
steps:
- uses: actions/checkout@v3
- run: npm install && npm run build
- uses: peaceiris/actions-gh-pages@v3
with:
github_token: ${{ secrets.GITHUB_TOKEN }}
publish_dir: ./dist
05

Votre site est en ligne ! 🎉

Après quelques secondes, votre application est accessible à l'adresse https://username.github.io/repo-name. Chaque nouveau push mettra à jour le site automatiquement.
